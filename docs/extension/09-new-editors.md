# Luna Toolkit — New Visual Editors (Batch 2)

> 11 new visual editors: Sprite Sheet/Animation, Tileset, Audio Mixer,
> Color Palette, Input Mapper, Timeline/Cutscene, Shader Preview,
> Font Preview, Localization, Physics Materials, World Map.
>
> Design principle: **one panel, one job, done well.**
> Reference: similar game engines' dedicated editors, pixel art tools's animation workflow,
> scripted event systems, integrated room/level editors.

---

## Shared Design Principles (Extended)

From batch 1: consistent toolbar / canvas / properties triptych.

**New rules for batch 2**:
- If a panel has no canvas, use a **master–detail** split instead
- Preview panels always have a real-time simulation (no static screenshots)
- Every panel's left side width ≤ 200px — don't crowd the workspace
- Toolbar max height: 36px (two rows if needed, never three)
- Status bar is always present: 1px top border, 24px height, dim text, key metrics
- All editors export Lua AND TOML unless the format is binary-only

## Editor 17: Sprite Sheet & Animation Editor

**Command**: `luna.editor.spriteAnim`
**One job**: Slice a sprite sheet into named frames and group them into animations.
**Reference**: pixel art tools (frame management), similar animation resource systems.

### Why this exists
Luna2D has a tilemap but no built-in way to define sprite animations. Users need
to manually write `{x=0, y=0, w=32, h=32}` tables. This editor generates them.

### Layout

```
┌───────────────────────────────────────────────────────────────────┐
│ SPRITE SHEET  File: [player.png ▼] [📂Load]                      │
│ Grid: W[16] H[16] [Auto-detect] Offset: X[0] Y[0] Pad: [0]      │
│                                                     [Export Lua] │
├────────────┬───────────────────────────────────────┬─────────────┤
│ ANIMATIONS │  SHEET VIEW                           │ FRAME DATA │
│            │                                       │            │
│ [+] [✕]    │   □ □ □ □ □ □ □ □   ← click to       │ Frame: 3   │
│            │   □ ▓ ▓ ▓ □ □ □ □     select/range    │ Row: 0     │
│ ▶ idle     │   □ □ □ □ □ □ □ □   shift+click to   │ Col: 2     │
│   (frames  │   □ □ □ □ □ □ □ □     extend range    │ X:  64     │
│    1–4)    │                                       │ Y:   0     │
│ ▶ walk     │  Colors per cell = tile id            │ W:  32     │
│   (frames  │  Selected frames highlighted in        │ H:  32     │
│    5–12)   │  accent color                         │            │
│ ▶ run      │                                       │ ANIMATION  │
│   (frames  │  ┌─  ANIMATION PREVIEW ──────────┐   │ walk       │
│    13–16)  │  │                               │   │ Frames:    │
│ ▶ jump     │  │   [animated sprite here]      │   │ 5,6,7,8,   │
│ ▶ fall     │  │   32×32 zoomed 4×             │   │ 9,10,11,12 │
│ ▶ attack   │  │                               │   │ FPS: [10]  │
│ ▶ die      │  │  ▶ [⏸] [⏮] [⏭]  Frame: 3/8  │   │ Loop: [☑]  │
│            │  └───────────────────────────────┘   │ Pingpong:  │
│            │                                       │ [ ]        │
│            │  Zoom: [-] [100%] [+]  Grid: [☑]     │            │
│            │  Show frame numbers: [☑]              │ [+ Frame]  │
│            │                                       │ [- Frame]  │
│            │                                       │ [↑ Move]   │
│            │                                       │ [↓ Move]   │

└───────────────────────────────────────────────────────────────────┘
```

### Export Format (Lua)
```lua
-- Generated by Luna Toolkit — Sprite Sheet Editor
return {
  image = "player.png",
  frameWidth = 32, frameHeight = 32,
  animations = {
    idle   = { frames = {1,2,3,4},        fps = 8,  loop = true },
    walk   = { frames = {5,6,7,8,9,10,11,12}, fps = 10, loop = true },
    run    = { frames = {13,14,15,16},    fps = 16, loop = true },
    jump   = { frames = {17,18},          fps = 6,  loop = false },
    fall   = { frames = {19,20},          fps = 6,  loop = false },
    attack = { frames = {21,22,23,24,25}, fps = 18, loop = false },
    die    = { frames = {26,27,28,29,30}, fps = 10, loop = false },
  }

```

Luna2D has a tile map but tiles are dumb IDs. This editor turns IDs into rich
objects with collision shapes, friction presets, tags, and autotile logic.

## Editor 20: Color Palette Editor

**Command**: `luna.editor.colorPalette`
**One job**: Define named color palettes that can be referenced across the game.
**Reference**: pixel art tools palettes, Lost Century palette workflows.

### Why this exists
Games need consistent color schemes. Named colors (`COL.PLAYER_BLUE`) are far
better than magic numbers scattered everywhere.

### Layout

```

│                                                     [Export Lua] │

│ │▓▓│▓▓│▓▓│▓▓│▓▓│▓▓│  │    [███████▓   ]   │   24 colors          │
│ ├──┼──┼──┼──┼──┼──┤  │ S: [80%]           │                      │
```

### Export (Lua)
```lua
-- Colors are exported as 0-1 range table with named keys
return {
  sky_blue   = {0.227, 0.482, 0.835, 1.0},
  fire_red   = {0.902, 0.133, 0.133, 1.0},
  shadow_dark = {0.059, 0.055, 0.071, 1.0},
  -- ...
}
```
IntelliSense: When `palette.lua` is loaded with `require("palette")`, the keys
appear in completion when calling `luna.graphics.setColor(palette.|)`.

## Editor 21: Input Mapper

**Command**: `luna.editor.inputMapper`
**One job**: Define named actions and bind them to keyboard keys, gamepad buttons, and mouse buttons.
**Reference**: similar game engines' input mapping editors, major engines' input managers, similar engines' key action systems.

### Why this exists
Hardcoded `luna.input.isDown("space")` scattered through the game is a maintenance
nightmare. Named actions (`input.isDown("jump")`) are portable and remappable.

### Layout

```

│  move_right      │  →          │  D          │ LS →      │  —    │
```

## Editor 22: Timeline / Cutscene Editor

**Command**: `luna.editor.timeline`
**One job**: Sequence timed events (camera moves, dialog triggers, sounds, spawns) into a cutscene.
**Reference**: similar game engines' animation players, event-based editors, cinematic timeline tools.

### Why this exists
Cutscenes in Lua are usually written as long coroutine chains with `wait()` calls.
This editor generates that code visually so designers can see timing at a glance.

### Layout

```
┌───────────────────────────────────────────────────────────────────┐
│ TIMELINE  Name: [intro_cutscene]  BPM: [120] for beat-snapping   │
│ [▶ Play] [⏸ Pause] [⏮ Reset]  Duration: [12.0s]   [Export Lua]  │
│ Snap: [0.25s ▼]  Zoom: [-][1×][+]  Loop: [ ]                    │
├────────────────┬──────────────────────────────────────────────────┤
│ TRACK TYPES    │  TIMELINE               sec:  0    2    4    6   │
│ [+ Add Track]  │  ─────────────────────────┼────┼────┼────┼────   │
│                │  🎥 Camera                │    ├────┤    ├──┤    │
│  ─ TRACKS ─    │  👤 Spawn: player         │─┤  │              │  │
│                │  👾 Spawn: enemy_1        │    │    ├──┤        │  │
│  🎥 Camera     │  💬 Dialog: npc_hello     │         ├────────┤  │  │
│  👤 Spawn      │  🔊 Music: battle.ogg     │  ├──────────────────┤│  │
│  💬 Dialog     │  ✨ Particles: explosion  │              ├──┤    │  │
│  🔊 Audio      │  🌙 Fade: black_in        │─────────────────┤  │  │
│  ✨ Effects    │  📷 Screenshot trigger    │                  ●   │
│  🌙 Fade       │                           │                      │
│  📷 Capture    │  ◄─────────────────── drag bar moves playhead ─► │
│  ⚙ Custom     │                                                   │
├────────────────┴──────────────────────────────────────────────────┤
│ SELECTED EVENT: Dialog: "npc_hello"                               │
│  Start: [3.5s]  End: [6.0s]  Speaker: [npc_wizard]               │
│  Text: ["Welcome, hero. The castle awaits..."]                    │
│  Condition: [player.hasQuest("main_quest") == true]               │
│  On finish: [setFlag("intro_seen", true)]                         │
├───────────────────────────────────────────────────────────────────┤
│ Track: Dialog  |  Duration: 2.5s  |  Events: 7  |  Playhead: 3.5s│
└───────────────────────────────────────────────────────────────────┘
```

### Export (Lua — as coroutine)
```lua
-- Generated by Luna Toolkit — Timeline Editor
return function(co_yield)
  -- Camera: pan to position (0, 0 → 400, 200)
  co_yield("camera_move", 0, 2.0, {from={0,0}, to={400,200}, ease="ease_in_out"})
  -- Dialog: npc_hello
  co_yield("dialog", 3.5, 6.0, {speaker="npc_wizard", text="Welcome, hero..."})
  -- Music
  co_yield("play_music", 0, 12.0, {file="battle.ogg", volume=0.8})
  -- Fade in from black
  co_yield("fade", 10.0, 12.0, {from=1, to=0})
end
```
