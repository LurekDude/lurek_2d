## Summary

The `i18n` module provides Lurek2D's internationalization and localization system, exposed to Lua scripts as `lurek.i18n.*`. Its job is to keep all user-facing text out of game source code and into locale-specific data files, making it straightforward to ship a game in multiple languages without touching game logic.

The central runtime type is `Catalog`, a two-level map from locale code (e.g. `"en"`, `"pl"`, `"ja"`) to dot-path key to translated string. A locale file is a TOML file with string key-value pairs supporting dot-path keys for nested namespacing (e.g. `"ui.button.ok"`). `Catalog::load_locale(code, path)` reads and parses a locale TOML file and merges it into the catalog under the given locale code. `Catalog::set_locale(code)` switches the active locale. `Catalog::tr(key)` performs a lookup in the active locale, falling back to the configured fallback locale (usually `"en"`) if the key is missing, and finally returning the raw key string if no translation is found at all. A `CatalogError` is returned for file parse failures.

`interpolation.rs` extends translation strings with named-variable substitution: `{variable_name}` placeholders in the translated string are replaced by values passed in a parameter map. `interpolate_pairs` takes an array of `(key, value)` pairs for ergonomic use from Lua.

`plural.rs` implements locale-aware plural rule selection, mapping a count integer to one of the CLDR plural form categories: `Zero`, `One`, `Few`, `Many`, `Other`. `pluralize_slavic` handles the non-standard Slavic plural rules (Polish, Russian, Ukrainian, etc.) where the `Few`/`Many` split depends on the last two digits of the count.

Minor additions to the `Catalog` API have improved key introspection and locale management for scripts that need to enumerate or programmatically validate available translations. Lua scripts accessing `lurek.i18n.*` benefit from these additions when implementing language-selection menus or automated localization coverage checks.

**Scope boundary**: Feature Systems tier. Depends on `runtime` (TOML loading). Lua bridge in `src/lua_api/i18n_api.rs`.

## Files

- `catalog.rs`: Implements `Catalog` storage, locale fallback handling, and typed catalog errors.
- `format.rs`: Pure-Rust number and date formatting (`format_number`, `format_date`, `locale_separators`, `days_to_ymd`, `month_name_tables`).
- `interpolation.rs`: Provides placeholder substitution helpers for localized strings.
- `mod.rs`: Declares the i18n submodules and re-exports the public catalog, interpolation, formatting, and pluralization APIs.
- `plural.rs`: Defines plural categories and locale-style plural selection helpers, including Slavic-specific rules.

## Types

- `CatalogError` (`enum`, `catalog.rs`): Typed error enum for missing locales, missing keys, and related catalog lookup failures.
- `Catalog` (`struct`, `catalog.rs`): The main locale-keyed translation store with fallback resolution and flat string lookup.
- `PluralForm` (`enum`, `plural.rs`): The plural-category enum used to select the correct localized string variant.

## Functions

- `Catalog::new` (`catalog.rs`): Creates an empty catalog.
- `Catalog::load` (`catalog.rs`): Loads or replaces a locale's flat string table.
- `Catalog::unload` (`catalog.rs`): Removes a locale from the catalog.
- `Catalog::has_locale` (`catalog.rs`): Whether a locale is loaded.
- `Catalog::locales` (`catalog.rs`): Returns all loaded locale codes.
- `Catalog::has_key` (`catalog.rs`): Whether a key exists in the currently active locale.
- `Catalog::keys` (`catalog.rs`): Returns all keys available in the active locale.
- `Catalog::get` (`catalog.rs`): Resolves a translation key using the active locale with fallback chain.
- `Catalog::translate` (`catalog.rs`): Like [`get`][Catalog::get] but returns the key itself when not found.
- `Catalog::set_key` (`catalog.rs`): Inserts or updates a single key in the given locale.
- `Catalog::export` (`catalog.rs`): Exports the given locale table as a flat `HashMap<String, String>`.
- `Catalog::merge` (`catalog.rs`): Merges key-value pairs into an existing locale without replacing the whole table.
- `Catalog::key_count` (`catalog.rs`): Returns the number of keys in the active locale.
- `Catalog::categories` (`catalog.rs`): Returns the unique first path-segments of all keys in the active locale.
- `Catalog::keys_in_category` (`catalog.rs`): Returns all keys in the active locale that start with the given category prefix.
- `Catalog::search` (`catalog.rs`): Simple substring search over values in the active locale.
- `Catalog::build_index` (`catalog.rs`): Builds an inverted word index for the active locale.
- `locale_separators` (`format.rs`): Returns `(decimal_separator, thousands_separator)` for the given locale code.
- `format_number` (`format.rs`): Formats `n` with `decimals` decimal places and locale-specific separators.
- `format_date` (`format.rs`): Formats a Unix timestamp (seconds UTC) as a locale-aware date string.
- `days_to_ymd` (`format.rs`): Converts days since Unix epoch to `(year, month, day)`.
- `month_name_tables` (`format.rs`): Returns (long_month_names, short_month_names) for English.
- `interpolate` (`interpolation.rs`): Interpolates `{key}` placeholders in `template` with values from `vars`.
- `interpolate_pairs` (`interpolation.rs`): Thin wrapper: accepts `&[(impl Display, impl Display)]` pairs as the variable map.
- `PluralForm::key` (`plural.rs`): Returns the canonical lowercase key string for this category.
- `PluralForm::english` (`plural.rs`): Returns the `PluralForm` for an English-style (one/other) count.
- `PluralForm::slavic` (`plural.rs`): Returns the `PluralForm` for a Slavic-style (one/few/many/other) count (closest to Russian/Polish).
- `PluralForm::from_key` (`plural.rs`): Parses a form key string to a `PluralForm`.
- `pluralize` (`plural.rs`): Selects the correct plural string from a form map for the given count.
- `pluralize_slavic` (`plural.rs`): Like [`pluralize`] but accepts integer counts and uses Slavic rules.
